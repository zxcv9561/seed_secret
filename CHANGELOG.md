## 🗓️ 패치별 상세 내역

### v1 — 기본 구조 (초기 구축)
- 로그인 시스템 (agent ID 기반)
- 핵심 섹션: 기관 소개 / 사건 일람 / 대상 보고서 / 요원 명부 / 작전 일지 / 기밀 문서 / 자유게시판
- 블록 에디터 (h1, h2, text, divider, image)
- 권한 시스템 3-tier (role / section perm / entity perm)
- 즐겨찾기 기능
- Supabase 연동

### v2 — 권한 & 관리
- 관리 콘솔 (master 전용)
- 섹션별 view/edit/delete 권한 설정
- 요원별 role (master / member / viewer)
- 공개/비공개/기밀 가시성 체계

### v3 — 메신저
- 1:1 실시간 채팅 (Supabase Realtime)
- 이모티콘 관리 + 전송
- 읽음 처리 + 안읽음 배지
- 대화 상대 목록 관리

### v4 — 미디어
- **주크박스**
  - URL (YouTube / SoundCloud) + 파일 업로드 지원
  - 재생 목록 기능 (생성/이름변경/삭제)
  - 드로어 닫아도 재생 유지 (iframe 보존)
- **자료실**
  - 파일 업로드 / 이미지 / 외부 링크 3가지 모드
  - 50MB 용량 제한
  - 그리드/리스트 뷰
  - 저장소 사용량 대시보드

### v5 — 스토리지 정리
- 엔티티 삭제 시 Storage 파일 자동 정리 (15+ 곳)
  - 게시글, 요원, 사건 등 블록 내 이미지
  - 요원 삭제 시 프로필 사진 + 이모티콘
  - 첨부파일, 주크박스 업로드

### v6 — 검색 기능
- 모든 섹션에 검색 도입
- IME 한글 입력 호환 (compositionstart/end)
- 제목 전용 / 제목+본문 토글
- 포커스 유지 (검색 후 재렌더링 시)

### v7 — 단톡방 🎊
- **그룹 채팅 시스템**
  - 메신저 대화 탭 → 1:1 / 그룹 서브탭 분리
  - 방 생성 모달 (이름 + 멤버 체크박스, 그룹별 정렬)
  - 방별 안읽음 카운트 (last_read_at 기반)
  - 발신자 이름 + 프로필 사진 표시 (연속 발신 시 생략)
- 메시지 선택 삭제 (⋮ 버튼 + 롱 프레스 500ms)
- 대화 전체 삭제 버튼
- 방 나가기 (생성자는 방 삭제)

### v8 — 게시판 고도화 1단계
- **좋아요 / 싫어요** (한 유저당 한 번)
  - ▲ / ▼ 토글 버튼 + 카운트
- **댓글 시스템**
  - 기본 댓글 작성/삭제
  - 프로필 사진 + 작성 일시 표시
  - Ctrl+Enter 빠른 등록
- **수정 모드 분리 (게시판 선행)**
  - 기본 읽기 모드, 수정 버튼으로 편집 모드 진입
  - master + 작성자 본인만 수정 가능
- 게시판 목록에 댓글/반응 카운트 표시

### v9 — 게시판 고도화 2단계 ✨
- **대댓글 (2단계 중첩)**
  - ↩ 답글 버튼
  - 들여쓰기로 계층 시각화
  - 답글 시 `@작성자` 자동 프리필
- **댓글 좋아요/싫어요**
- **댓글 수정 기능**
  - 본인 댓글 + master만 가능
  - `(수정됨)` 뱃지 표시
- **@멘션 기능**
  - 실존 요원명만 인식하여 하이라이트
  - 멘션 시 자동 알림 발송
- **실시간 동기화**
  - 4개 테이블 Realtime 구독
  - 타인의 좋아요/댓글 즉시 반영
- **알림 시스템**
  - 메신저에 "알림" 탭 추가
  - 4가지 타입: comment / reply / reply_on_post / mention
  - 안 읽음 좌측 빨간 줄 표시
  - 클릭 시 해당 게시글로 자동 이동 + 읽음 처리
  - "모두 읽음" 일괄 처리

### 최종 — 전 섹션 수정 모드 통일 🎯
- 게시판에 있던 수정/완료 버튼 패턴을 **모든 상세 페이지**에 적용
  - 기관 소개, 사건 일람, 대상 보고서, 요원 명부, 작전 일지, 기밀 문서
- 공통 헬퍼 함수 추가
  - `editToggleButtons(canModify)`: 수정/완료 버튼 생성
  - `bindEditToggleButtons(hdr)`: 이벤트 바인딩
  - `editModeBanner()`: 편집 중 배너
- 기본: 읽기 모드 (안전)
- 수정 버튼 클릭 → 노란 배너 + 편집 가능
- 완료 버튼 → 읽기 모드 복귀
- 페이지 이동 시 자동 리셋

## 🐛 주요 버그 픽스

### Promise.all → Promise.allSettled
**증상**: 한 테이블 로드 실패 시 주크박스/대화방/게시판 전체가 안 보임
**원인**: v8/v9 SQL 미실행 시 전체 throw
**해결**: 개별 실패 허용, 빈 배열로 fallback

### 409 Conflict (단톡방 생성 실패)
**증상**: 방 만들 때 409 에러 → 멤버 insert 실패 → 방 목록 비어있음
**원인**: 
1. 중복 키 충돌 (insert 시 FK 위반)
2. 비동기 순서 문제 (방 저장 전 멤버 insert 시도)

**해결**:
1. `insert` → `upsert` (onConflict: 'room_id,agent_id')
2. 방을 먼저 `await insert` → 그 다음 멤버 `await upsert`
3. 멤버 실패 시 방 롤백

### 주크박스 빈 화면
**증상**: 드로어 열어도 내용 비어있음
**원인**: `.jb-playlist-tabs` CSS 중복 정의 + `flex-shrink` 누락
**해결**: 중복 제거, `min-height: 0` 추가, `minmax(0, max-content)` grid

### 단톡방 이모티콘 과대 표시
**증상**: 그룹 채팅에서 이모티콘이 원본 크기로 거대하게 출력
**원인**: grid의 `1fr`이 이미지를 가용 공간만큼 확장
**해결**: `minmax(0, max-content)` + `!important`로 160x160 강제

### 대화방 새로고침 시 사라짐
**증상**: DM 대화 상대 목록이 새로고침 후 사라짐
**원인**: `chatPartnersCache`가 메모리에만 저장됨
**해결**: `saveUIState()`에 포함시켜 localStorage 영속화

### 주크박스 URL 재생 유지
**증상**: 드로어 닫았다 열면 YouTube 처음부터 재생
**원인**: 매번 `renderJukebox`가 `body.innerHTML = ...`로 iframe 파괴
**해결**: 드로어 토글 시 재렌더링 건너뛰기 (`.jb-playlist-tabs` 존재 체크)

## 💡 사용 팁

**수정 모드**
- 모든 상세 페이지는 기본 읽기 모드
- 우측 상단 "● 수정" 버튼으로 편집 진입
- "● 완료" 버튼 또는 페이지 이동 시 자동 종료

**멘션**
- `@요원이름` 형식 (띄어쓰기 주의)
- 실존 이름만 알림 발송됨

**단톡방**
- 메신저 → 대화 탭 → "그룹" 서브탭
- `+ 새 단톡방` 버튼으로 생성
- 방장이 방 삭제 시 모든 멤버에게서 사라짐
- 일반 멤버가 나가면 본인만 제외됨

**알림 관리**
- 메신저 프로필 탭에서 뮤트 설정
- 뮤트 시 알림음 안 남 + 배지 숨김
- 최근 100개만 저장
